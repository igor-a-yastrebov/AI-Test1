## Сводка изменений (2026-06-27)

Требование `C-8` изменено: кнопка SQRT теперь должна вычислять **кубический** корень (вместо квадратного).
В `calcEngine.ts` реализован `Math.sqrt` — необходимо заменить на `Math.cbrt`.
Существующие e2e-тесты для SQRT проверяют результаты квадратного корня — нужно обновить ожидаемые значения.

---

### TASK-4 ✅
**Требование:** C-8
**Тип:** `реализация`
**Описание:** В функции `reduce` в ветке `case 'SQRT'` заменить `Math.sqrt(arg)` на `Math.cbrt(arg)`. Кубический корень определён для отрицательных чисел, поэтому убрать проверку `if (arg < 0) return { ...INITIAL_STATE, display: 'Ошибка' }` — она более не нужна.
**Файлы:** `src/domain/calcEngine.ts`
**Критерий приёмки:** `reduce(state, { type: 'SQRT' })` для числа 27 возвращает `display: '3.00'`; для 8 возвращает `display: '2.00'`; для −8 возвращает `display: '-2.00'` (а не 'Ошибка').

---

### TASK-5 ✅
**Требование:** C-8
**Тип:** `тест`
**Описание:** Обновить e2e-тесты в spec-файле для SQRT под кубический корень: (1) ввести 27, нажать SQRT — табло показывает `3.00`; (2) ввести 8, нажать SQRT — табло показывает `2.00`; (3) ввести 0, нажать SQRT — табло показывает `0.00`; (4) убедиться, что для отрицательного результата промежуточной операции (например, 0 − 8 =, затем SQRT) табло показывает `-2.00`, а не 'Ошибка'.
**Файлы:** `e2e/calc-sqrt.spec.ts`
**Критерий приёмки:** Все обновлённые тест-кейсы проходят при запуске `npx playwright test`.

Конвейер из Агентов:
* PM - дает задание другому агенту. Например: BA проанализируй вот такую идею и выдай Requirements Increment. Анализирует инкремент и принимает финальное решение об изменениях. Другой пример: ARCHITECT, сделай такое-то изменение в архитектуре и дай задачу DEVELOPER'у на необходимые изменения в коде.
* BA - анализирует бизнес-потребности и переводит их в требования.
* ARCHITECT - создает архитектуру приложения и другие правила, которым должен следовать код
* DEV-TASKS - тех. аналитик, отлавливает изменения в требованиях и создает для них задачи для DEVELOPER
* DEVELOPER - реалирует код по файлу DEV-Tasks, создает unit tests, etc. Комитит работающий код

# GENERAL - общие требования
G-1. desktop application for Windows
G-2. tech stack: node.js + typescript
G-3. Detailed technical architecture. Described in Architecture.md file
# UI - требования к интерфейсу пользователя
UI-1. верхняя часть экрана - табло с цифрами. Табло отображает текущее число с дробной частью. Количество значащих цифр - до 10, цифр после запятой - до 3.
UI-2. под таблом располагаются активные кнопки:
* цифры от 0 до 9
* математические операции: сложить, вычесть, умножить, разделить
* C - очистить
* = - вычислить результат текущей операции
* . - ввод десятичной точки (UI-3)
# UI - допущения при реализации
UI-3. кнопка "." позволяет ввести дробную часть числа; повторное нажатие "." при уже введённой дробной части игнорируется.
UI-4. если длина числа на табло превышает 7 символов, размер шрифта автоматически уменьшается, чтобы число помещалось на экране.
# CALC - требования на вычисления
C-1. при нажатии на кнопку C, табло должно очиститься и показать число 0.00
# CALC - допущения при реализации
C-2. ввод числа: цифры набираются последовательно. Ведущий ноль заменяется цифрой. Максимум 10 цифр в целой части и 3 цифры после запятой — лишние нажатия игнорируются.
C-3. при нажатии на оператор (+, −, ×, ÷) текущее число сохраняется как первый операнд; следующий ввод начинает второй операнд.
C-4. цепочка операций: если оператор нажат повторно (ещё не нажато =), промежуточный результат вычисляется немедленно и становится новым первым операндом. Например: 2 + 3 × 4 = → (2+3)×4 = 20.
C-5. нажатие = без введённого второго операнда (т.е. сразу после оператора) не выполняет вычисление.
C-6. деление на ноль: на табло отображается "Ошибка", состояние калькулятора сбрасывается.
C-7. результат вычисления форматируется: минимум 2 знака после запятой, максимум 3. Пример: 5 → 5.00, 1.5 → 1.50, 3.141 → 3.141. Если результат выходит за пределы 10 значащих цифр (≥ 10^10), отображается "Ошибка".
